## 📌 Project Overview

This case study explores the sales and profitability data of a beverage company across multiple U.S. states. Using **Microsoft SQL Server (SSMS)**, I answered **29 real business questions** covering aggregations, joins, window functions, stored procedures, user-defined functions, transactions, and more.

The goal was to extract meaningful business insights from raw data — identifying profitable states, top-performing products, marketing spend, and sales trends.

---

## 🗂️ Dataset Overview

The dataset consists of **3 tables:**

| Table | Description |
|-------|-------------|
| `fact` | Core transactional table — sales, profit, COGS, marketing, inventory, expenses |
| `Product` | Product details — Product ID, Product Name, Product Type |
| `Location` | Geographic data — Area Code, State |

**Key Columns:**
- `fact` → `Date`, `ProductId`, `Area_Code`, `Sales`, `Profit`, `COGS`, `Marketing`, `Inventory`, `Total_Expenses`, `Budget_Margin`
- `Product` → `ProductId`, `Product`, `Product_Type`, `Type`
- `Location` → `Area_Code`, `State`

## 📋 Business Questions Solved

| # | Business Question | Concepts Used |
|---|-------------------|---------------|
| 1 | Number of distinct states in Location table | `COUNT DISTINCT` |
| 2 | Count of Regular type products | `WHERE`, `COUNT` |
| 3 | Total marketing spend for Product ID 1 | `SUM`, `WHERE` |
| 4 | Minimum sales across all products | `MIN` |
| 5 | Maximum Cost of Goods Sold (COGS) | `MAX` |
| 6 | Products where type is Coffee | `WHERE` filter |
| 7 | Records where Total Expenses > 40 | Conditional filter |
| 8 | Average sales in Area Code 719 | `AVG`, `WHERE` |
| 9 | Total profit generated by Colorado | `SUM`, `INNER JOIN` |
| 10 | Average inventory per Product ID | `GROUP BY`, `AVG` |
| 11 | States listed in sequential order | `DISTINCT`, `ORDER BY` |
| 12 | Products with avg Budget Margin > 100 | `GROUP BY`, `HAVING` |
| 13 | Total sales on 2010-01-01 | `SUM`, date filter |
| 14 | Avg Total Expense per Product per Date | Multi-column `GROUP BY` |
| 15 | Combined view of all key attributes | Multi-table `JOIN` |
| 16 | Sales-wise ranking without gaps | `DENSE_RANK()` |
| 17 | State-wise total profit and sales | `JOIN`, `GROUP BY` |
| 18 | State + Product wise profit & sales with subtotals | `ROLLUP`, `COALESCE` |
| 19 | Projected sales after 5% increase | Calculated column |
| 20 | Maximum profit per Product ID and Type | `MAX`, `JOIN`, `GROUP BY` |
| 21 | Stored procedure to filter by Product Type | `CREATE PROCEDURE` |
| 22 | Classify records as Profit or Loss | `CASE WHEN` |
| 23 | Weekly sales totals with subtotals | `DATEPART`, `ROLLUP` |
| 24 | Unique and common Area Codes across tables | `UNION`, `INTERSECT` |
| 25 | User-defined function to filter Product Type | `CREATE FUNCTION` |
| 26 | Update Product Type and safely undo it | `UPDATE`, `TRANSACTION`, `ROLLBACK` |
| 27 | Records where Total Expenses between 100–200 | `BETWEEN` |

## 💡 Key Insights

- **Colorado** generated significant total profit, identified via state-level JOIN aggregation.
- **Product ID 1** had measurable marketing spend — useful for evaluating marketing ROI.
- Several products had an **average Budget Margin exceeding 100**, indicating healthy margin performance.
- Using `DENSE_RANK()`, products were ranked by sales **without gaps** — helpful for competitive analysis.
- Weekly `ROLLUP` analysis revealed **sales patterns across different weeks**, useful for seasonal planning.
- The `CASE WHEN` classification showed which transactions fell into **profit vs loss zones** based on expense thresholds.
- `UNION` and `INTERSECT` on Area Codes confirmed **data consistency** between the fact and Location tables.

## ▶️ How to Run

1. Open **SQL Server Management Studio (SSMS)**
2. Connect to your local SQL Server instance
3. Open `Beverage_Sales_Profitability_Analysis.sql`
4. Run the `CREATE DATABASE` and `USE Beverage` statements first
5. Import `beverage_data.csv` into the respective tables (`fact`, `Product`, `Location`)
6. Execute queries one by one or section by section
